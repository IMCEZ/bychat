# 开发步骤手册

本目录把 BACKLOG.yaml 中的任务拆成可以手把手执行的步骤。

每个步骤字段含义：

- do：具体要做什么
- tech：使用的技术
- touch：本步允许改动的文件
- do_not_touch：本步禁止改动的范围，明确边界
- boundary：本步的作用范围界限
- done_effect：本步完成后的可见效果
- self_check：用最少 token 完成的自检方式，优先用构建和小范围 grep

## 迭代顺序

- STEPS-I01.yaml　protocol-v2 骨架　（当前）
- I02　可靠网络会话（待拆分）
- I03　安全身份登录（待拆分）
- I04　数据库 v2 与迁移（待拆分）
- I05　增量同步闭环（待拆分）

## 执行原则

- 一次只做一个步骤，做完立即构建自检再提交
- 每步保持应用可构建、现有功能不回退
- 用户可见文本与文档一律中文
